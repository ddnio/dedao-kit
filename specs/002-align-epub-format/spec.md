## 澄清事项

### 会话 2025-12-10
- Q: 如何处理 HTML 结构和 CSS 的差异？ → A: **完全模仿参考结构**：将 HTML 结构调整为与参考文件一致（例如使用 `<div class="header0">`、`<div class="part">` 包裹内容，将脚注 `<aside>` 放在段落附近或特定容器中），并复制其 CSS 类名。
- Q: 如何处理样式表的位置？ → A: **移动到子目录**：将生成的 `style.css` 移动到 `EPUB/css/` 目录（并可能重命名为 `cover.css` 或与参考一致的名称），并更新 XHTML 中的引用路径。
- Q: 如何处理图片命名和标签属性？ → A: **严格对齐命名与标签**：调整代码以生成 `image_XXX.ext`（3位填充）格式的文件名，并在 `<img>` 标签中使用与参考一致的属性（如 `class="epub-footnote ..."`，去除不必要的小数位）。
- Q: 是否只需对齐第一章还是整个 EPUB？ → A: **全书一致**：整个 EPUB 的结构和样式都要与参考保持一致，第一章只是主要验证点。

## 用户场景与测试 *(必填)*

### 用户故事 1 - 一致的视觉排版 (优先级: P1)

作为一名用户，我希望浏览器扩展生成的 EPUB 文件具有与 Go 命令行工具生成的 EPUB 文件相同的视觉排版和样式，以便我在不同平台间获得一致的阅读体验。

**优先级理由**: 用户明确报告了排版和格式问题，并指出这与现有解决方案相比是一个缺陷。

**独立测试**: 可以通过使用扩展生成一本 EPUB 电子书，并将“第一章”与提供的参考文件 `@131902_庄子_庄子 著；孙雍长 译注.epub` 进行并排对比来测试。

**验收场景**:

1. **假设** 拥有 Go 项目生成的参考 EPUB，**当** 我使用浏览器扩展生成同一本书时，**那么** 第一章的字体大小、页边距、行高和段落间距在视觉上应与参考文件匹配。
2. **假设** 内容中包含脚注或图片等特定元素，**当** 在 EPUB 阅读器中渲染时，**那么** 它们的位置和样式应与参考实现保持一致。

---

### 用户故事 2 - 结构对齐 (优先级: P2)

作为一名开发者/维护者，我希望内部 HTML 结构和 CSS 定义与 Go 项目的标准保持一致，以便两个工具之间的后续维护和调试保持一致。

**优先级理由**: 确保长期的可维护性，减少同一逻辑的两个实现之间的“偏移”。

**独立测试**: 检查生成的 EPUB 文件内部的 `OEBPS/Styles/` (或等效目录) 和 `OEBPS/Text/` 文件。

**验收场景**:

1. **假设** 生成的 EPUB 文件，**当** 我检查 CSS 文件时，**那么** 它应包含与参考 EPUB 相同的样式定义（类、规则）。
2. **假设** 生成的章节 HTML，**当** 我检查标记时，**那么** 它应使用与参考文件完全匹配的标签结构和类名（例如 `div.header0`，`div.part`，`aside` 的位置）。

### 边缘情况

- **缺少参考样式**: 如果 Go 项目使用了静态参考文件中不存在的动态样式，扩展应优雅地回退到可读的默认样式。
- **不同的内容类型**: 如果书籍包含第一章中不存在的元素（例如，复杂的表格、代码块），它们的样式仍应与参考文件的一般设计语言保持一致。

## 需求 *(必填)*

### 功能需求

- **FR-001**: 系统**必须**使用复制自 Go 项目 EPUB 输出的样式规则的 CSS 样式表，包括文件位置 (`EPUB/css/cover.css`)。
- **FR-002**: 系统**必须**生成与 Go 项目参考文件完全一致的 HTML 结构，包括使用 `<div class="header0">`、`<div class="part">` 等容器。
- **FR-003**: 系统**必须**专门解决用户指出的“第一章”中的排版问题，确保标题和正文的嵌套结构与参考文件一致。
- **FR-004**: 系统**必须**确保特殊元素（脚注、图片、说明文字）遵循参考文件的布局模式，特别是脚注 `<aside>` 标签的放置位置。
- **FR-005**: EPUB 内部目录结构（例如，图片、样式的位置）**应该**与参考文件对齐，以最大限度地减少结构差异。
- **FR-006**: 系统**必须**确保图片资源命名格式为 `image_XXX.ext`（3位数字填充），并在 `<img>` 标签中包含正确的 `class` 和 `alt` 属性，与参考文件一致。
- **FR-007**: 系统**必须**确保生成的整个 EPUB（所有章节及资源）在结构与样式上与参考文件保持一致，以避免后续章节产生偏差。

### 关键实体

- **样式表**: 包含排版和布局规则的 CSS 文件定义。
- **章节文档**: 代表单个章节的 XHTML 文件结构。
- **EPUB 包**: 包含清单和资源的整体 ZIP 结构。

## 成功标准 *(必填)*

### 可衡量的结果

- **SC-001**: 扩展生成的 EPUB 与参考 EPUB 的“第一章”视觉对比显示无明显排版差异（文本流、字体大小或元素位置的差异极小）。
- **SC-002**: 生成的 EPUB 中的 CSS 文件包含参考文件中存在的关键样式规则。
- **SC-003**: 生成的 EPUB 通过标准验证（例如 epubcheck），且未因格式更改引入新错误。
- **SC-004**: 参考 EPUB 与扩展生成的整本书在结构和样式一致性验证中通过检查，且对比差异限于可忽略级别。
