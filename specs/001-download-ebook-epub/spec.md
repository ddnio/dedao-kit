## 用户场景与测试 *(mandatory)*

### 用户故事 1 - 电子书下载 (优先级: P1)

作为得到 Web 端已登录用户，我希望能够将已购买的电子书下载为 EPUB 格式，以便在本地阅读器中进行离线阅读。

**优先级理由**: 这是本功能的 MVP 核心需求，直接交付用户价值（离线阅读）。

**独立测试**: 
用户登录得到 Web 版，打开一本已购电子书的详情页或阅读页，点击扩展程序的下载按钮，能够成功保存一个有效的 `.epub` 文件到本地，且该文件能被标准阅读器（如 iBooks, Calibre）打开。

**验收场景**:

1. **Given** 用户已登录得到 Web 端并拥有某本电子书的阅读权限，**When** 用户点击扩展程序的“下载 EPUB”按钮，**Then** 扩展程序显示下载进度，并在完成后自动触发浏览器文件下载，保存名为`[书名].epub`的文件。
2. **Given** 用户未登录或未购买该书，**When** 用户尝试下载，**Then** 系统应提示错误信息（如“未授权”或“请先购买”），不产生下载文件。
3. **Given** 网络中断或请求超时，**When** 下载过程中发生错误，**Then** 系统应提示“下载失败”并允许重试。

### 边缘情况

- **未登录状态**: 用户在未登录状态下点击下载，应提示登录。
- **未购买内容**: 尝试下载试读或未购书籍，应被拦截并提示。
- **特殊字符**: 书名包含文件系统非法字符（如 `/`, `:`, `*`）时，文件名应自动清洗。
- **大文件**: 书籍包含大量图片导致生成时间过长，应有加载/进度提示，避免用户认为卡死。

## 需求 *(mandatory)*

### 功能需求

- **FR-001**: 系统必须能够从当前页面 URL 或页面内容中提取电子书的唯一标识符 (ID)。
- **FR-002**: 系统必须复用当前浏览器会话的 Cookies 进行 API 请求，通过得到服务端的鉴权。
- **FR-003**: 系统必须根据电子书 ID 调用 API 获取读书令牌（ReadToken）。
- **FR-004**: 系统必须根据 ReadToken 获取书籍元数据（标题、作者、封面图、章节列表及目录结构）。
- **FR-005**: 系统必须顺序逐章获取电子书章节内容（SVG格式，Base64编码+AES-256-CBC加密）。
- **FR-006**: 系统必须在浏览器中解密章节内容（使用硬编码密钥和IV进行AES-256-CBC解密）。
- **FR-007**: 系统必须将解密后的SVG内容转换为HTML格式（参考Go项目的SVG→HTML转换逻辑）。
- **FR-008**: 系统必须按照 EPUB 3.0 标准使用jszip库生成 `.epub` 文件（包括文件头、目录、章节HTML、资源等）。
- **FR-009**: 系统必须在下载过程中提供可视化的状态反馈（如：获取元数据中、下载章节 X/N、生成EPUB中）。
- **FR-010**: 系统必须对单个章节内容获取失败实现自动重试机制（最多重试3次）。
- **FR-011**: 系统必须拦截并处理 API 错误（如 401 未授权, 403 禁止访问），向用户展示友好的错误信息。
- **FR-012**: 下载完成后，系统必须自动触发浏览器文件下载，保存为 `[书名].epub` 文件。

### 关键实体

- **EbookInfo**: 包含书籍 ID、标题、作者、简介、封面 URL。
- **Chapter**: 包含章节 ID、标题、正文内容（HTML/Markdown）、排序索引。
- **DownloadTask**: 包含当前下载任务的状态（Pending, Downloading, Processing, Completed, Failed）和进度。

## 成功标准 *(mandatory)*

### 可衡量的结果

- **SC-001**: 95% 以上的已购书籍下载任务在网络正常情况下能成功生成有效的 EPUB 文件。
- **SC-002**: 生成的 EPUB 文件通过标准 EPUB 验证工具（如 epubcheck）无严重错误（Critical Errors）。
- **SC-003**: 对于纯文字类书籍（< 500章），从点击下载到文件保存的平均处理时间不超过 30 秒。
- **SC-004**: 用户界面在 API 请求期间保持响应，进度条或状态提示实时更新，无假死现象。

- Q: 用户认证与访问方式 → A: 浏览器扩展 + HTML 弹窗（复用浏览器Cookies认证）

- Q: SVG内容处理与EPUB生成库 → A: 参考Go项目实现，SVG→HTML转换，使用jszip + 手工EPUB结构避免小众库风险
